# Standard for contributing

### Following the [Conventional Commits](https://www.conventionalcommits.org/pt-br/v1.0.0/) but adapting it on to our own necessities

#### Always describe your commit using the following:

- `feat:` Use it when adding new features

- `docs:` Use it when adding/changing documentation

- `refactor:` Use it when changing code that already exists

- `rename:` Use it when renaming any files

- `remove:` Use it when removing any files

## Examples

`feat(main.c): Adding determinant function`

`docs(contributing.md): Adding examples for commits`

`refactor(determinant.c): Changing if's condition using AND`

`rename(determinant.c): Changing determinant.c to det.c`

`remove(det.h): Now using determinant.h`
